# ERRATA (v0.4 → v0.5)

This document lists what was wrong in the original manuscript / code and how each
item was corrected, with the empirical evidence. All numbers come from
[`experiments/verification/verify.py`](experiments/verification/verify.py)
(full log: [`experiments/verification/results.txt`](experiments/verification/results.txt)),
run with **real primes** (segmented sieve) and the **real zeros** in `data/zeros.csv`.

The original manuscript is preserved verbatim as `spectral_primes_v1_original.docx`.
The corrected manuscript is regenerated by `scripts/build_manuscript.py`
(`spectral_primes.docx` + `MANUSCRIPT.ru.md`).

---

## 1. Phase and sign of the operator — **WRONG → fixed**

**Was.** `U(x,Λ) = Σ ω(γ,Λ) cos(γ ln x)/γ`, justified by "φ = arg(ρ) = π/2, which
corresponds to φ = 0". High `U` was claimed to mark high prime density.

**Problem.** The explicit formula gives two different objects, neither equal to `+cos/γ`:

- *Counting* function: `(ψ(x)−x)/√x ≈ −2 Σ [(½)cos(γ ln x) + γ sin(γ ln x)]/(¼+γ²)`,
  i.e. `≈ −2 Σ sin(γ ln x)/γ` — phase is **sin**, weight `1/γ`.
- *Density* (this is what window counts measure): differentiating,
  `ψ′(x) = 1 − 2 x^{−1/2} Σ_{γ>0} cos(γ ln x)` — phase is **cos**, **no** `1/γ`, and
  primes are denser where `Σ cos(γ ln x)` is **negative**.

So the correct density operator is `D(x,Λ) = −Σ ω(γ,Λ) cos(γ ln x)` (note the leading
minus, so high `D` ⟺ more primes).

**Evidence** (Λ=200, 79 zeros, 2·10⁵ windows, x∈[2·10⁶,9·10⁶]):

| operator | Pearson r | Z = r√n | high-op density | low-op density |
|---|---|---|---|---|
| `+cos/γ` (original) | **−0.0306** | **−13.7** | 6462.5 | 6506.4 |
| `sin/γ` (counting) | +0.0088 | +3.9 | 6477.9 | 6490.3 |
| `−cos` = D (density) | **+0.0591** | **+26.4** | 6558.3 | 6424.2 |

The original operator **anti-correlates** with density (its "maxima" are prime-*poor*):
the sign was backwards. The corrected `D` correlates positively and significantly.

**Fix.** [`operator.py`](src/spectral_primes/operator.py) keeps `U_batch` for provenance
(with a warning) and adds `U_density_batch` (correct) and `U_psi_batch` (counting phase).

## 2. "Effect" is real but microscopic — **overstated → quantified**

**Was.** Abstract led with "Z = 4.22, P = 1.2·10⁻⁵", framed as a strong fingerprint.

**Reality.** Even with the *correct* operator the correlation is `r ≈ 0.059`, i.e.
`R² ≈ 0.0035` — the operator explains ~0.35% of the variance of window density. This is
exactly what theory predicts: the density fluctuation has amplitude `O(x^{−1/2})`. The
effect is statistically real (large Z because n is large) but predictively negligible.
The original `Z = 4.22 / P = 1.2·10⁻⁵` at `x ~ 10¹⁴` is **not reproducible** from the
repository (no data, no code path) and is removed.

## 3. `|S| = O(√N log N)` — **tautological → correctly attributed**

**Was.** Presented as a property of the variance-selection criterion, "verified" by a
table (|S| ≈ 42, 138, 421, 1247; R² = 0.9997) that the repo cannot reproduce.

**Reality.** With `Λ = c√N`, the number of zeros below the cutoff is `N(Λ) ≈
(Λ/2π)ln(Λ/2π) = O(√N log N)` by the Riemann–von Mangoldt counting formula. The
selection keeps a subset of those, so `|S| = O(√N log N)` is essentially the zero count,
not a discovery about the criterion. Measured (x₀=5·10⁶, δ=300, n_t=64):

| N | Λ=c√N | #γ≤Λ | \|S\| | RvM N(Λ) |
|---|---|---|---|---|
| 100 | 28.3 | 3 | 1 | 3.1 |
| 800 | 80.0 | 21 | 6 | 20.5 |
| 2400 | 138.6 | 47 | 10 | 47.0 |

`#γ≤Λ` matches `N(Λ)` to integer precision. With **Λ fixed at 80**, `|S|` saturates at 6
(extra zeros are above the cutoff, ω≈0) — there is no growth law at all. The criterion is
effectively a low-pass filter on γ.

## 4. Speedup claim (15–20% sieve reduction) — **FALSE → retracted**

**Was.** "Pre-marking resonance zones reduces the sieve region by 15–20% with ≤2% loss
of candidates" (motivating MPQS/GNFS use).

**Test.** Rank 4·10⁴ windows by the operator, scan from high to low, measure captured
primes vs fraction scanned:

| scan top | D (−cos) | +cos/γ | oracle (true density) | random |
|---|---|---|---|---|
| 10% | 9.98% | 10.10% | 11.11% | 10.00% |
| 20% | 20.01% | 20.13% | 21.77% | 20.01% |
| 50% | 50.10% | 49.85% | 52.50% | 50.00% |

Capture ≈ the uniform baseline. To lose `<2%` of primes you can skip only **~2%** of
windows — identical to a **random** ordering, not 15–20%. Even a perfect oracle on the
true density only reaches 11.1% capture in the top 10%, so the ceiling is tiny. The
applied speedup claim is **not supported and is withdrawn**.

## 5. Permutation test — **degenerate null clarified**

Shuffling the multiset `{γ}` leaves both the plain sum and the density operator
invariant (same set of frequencies). The original "p < 10⁻⁴ over 10⁴ permutations"
referred to a degenerate construction. The code already guards this
(`allow_degenerate_null` / `rank_anneal`); the manuscript now states it plainly.

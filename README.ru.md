# Spectral primes

**Язык:** [English](README.md) · Русский

Небольшой **набор инструментов на Python** для экспериментов со *спектральной* конструкцией, связанной с мнимыми частями `γ_n` нетривиальных нулей дзета-функции Римана. Это **вычислительное приложение** к препринту *«Спектральная аппроксимация локальных максимумов плотности простых чисел»* (исходный текст: `spectral_primes.docx` в корне репозитория).

Это ПО **не** доказывает утверждения из того документа. Оно воспроизводит **определения и схемы расчёта**, чтобы запускать их на обычном компьютере, развивать их и смотреть поведение на умеренных масштабах (например, `x ~ 10^6–10^8`). В препринте зарегистрированный Monte Carlo использует гораздо большие `x` и намного больше нулей; **не ожидайте здесь тех же величин эффекта и p-value**.

## Контекст

Явная формула фон Мангольда связывает распределение простых с нулями `ρ = ½ + iγ` функции `ζ(s)`. Упрощённый *осциллирующий* объект из препринта — **регуляризованный спектральный оператор** (фаза `φ_n ≡ 0`, как в рукописи):

$$
U(x,\Lambda) = \sum_{\gamma_n \le \Lambda} \omega(\gamma_n,\Lambda)\,\frac{\cos(\gamma_n \ln x)}{\gamma_n}
\qquad
\omega(\gamma,\Lambda) = \exp\left(-\frac{\gamma^2}{2\Lambda^2}\right).
$$

В сумму входят только нули с `γ_n ≤ Λ`. В Python `U` считается **векторизованно** для многих `x` (`numpy`). Дополнительно есть **C++**-утилита [`spectral_u`](cpp/README.md) на **GNU MPFR** (и GMP) для произвольной точности той же суммы — удобно при больших `x` и для сверки.

## Интерфейс командной строки

Общие опции (до подкоманды):

| Опция | По умолчанию | Смысл |
|--------|----------------|--------|
| `--csv` | `data/zeros.csv` | Путь к CSV с нулями |
| `--sqlite` | `data/zeta_zeros.sqlite3` | Используется, если файл существует |

### `demo` — полный `U`, три группы

Сравнивает среднюю плотность простых (на `10^5` целых) в окнах с центрами при **высоком** `U`, **случайных** центрах и **низком** `U` относительно среднего ± `sigma_u`·СКО `U` на референсной сетке в `[x_lo, x_hi]`.

| Опция | По умолчанию | Описание |
|--------|----------------|----------|
| `--Lambda` | `80` | Обрезка `Λ` |
| `--x-lo`, `--x-hi` | `2_000_000`, `8_000_000` | Диапазон выборки центров |
| `--half-width` | `400` | Половина ширины окна подсчёта простых |
| `--n` | `40` | Размер каждой группы |
| `--sigma-u` | `1.5` | Порог в единицах референсного СКО |
| `--seed` | `42` | Seed ГПСЧ |

### `demo-sparse` — `U_sparse` + подмножество Varₙ

Тот же трёхгрупповой дизайн, но вместо `U` используется **U_sparse** (см. ниже). Квадраты вкладов усредняются на `[x − δ, x + δ]` по `n_t` точкам.

| Опция | По умолчанию | Описание |
|--------|----------------|----------|
| `--delta` | `300` | Половина интервала `δ` для локальной энергии |
| `--n-t` | `48` | Число точек по времени |
| `--rank-anneal` | `0` | Слотовый множитель `exp(rank_anneal·(slot+1)/N))`; `0` = правило только по γ |
| *(как у `demo`)* | | `Lambda`, `x-lo`, `x-hi`, `half-width`, `n`, `sigma-u`, `seed` |

### `permute` — перестановочный нуль для контраста sparse

Фиксируется пул центров и **фиксированная** случайная контрольная группа **B**. Перемешивается **упорядоченный** вектор `γ` (Fisher–Yates). Статистика: `средняя плотность | A_высокий) − средняя | B)`. Выводится двусторонняя **эмпирическая** p-value.

| Опция | По умолчанию | Описание |
|--------|----------------|----------|
| `--reps` | `200` | Число перестановок |
| `--pool-factor` | `80` | Размер пула ≥ `n * pool_factor` |
| `--rank-anneal` | `0.05` | Должно быть `> 0` для невырожденного нуля; `0` только с `--allow-degenerate-null` |
| `--allow-degenerate-null` | выкл. | Разрешить `rank_anneal=0` (вырожденный нуль; только диагностика) |
| *(пересечение с `demo-sparse`)* | | |

### `curve` — выгрузка `U(x)` (полный оператор)

Логарифмическая сетка по `x`; используется **полный** `U`, не sparse.

| Опция | По умолчанию | Описание |
|--------|----------------|----------|
| `--x-lo`, `--x-hi` | `1e6`, `1e7` | Диапазон (float) |
| `--points` | `50` | Число точек |
| `--Lambda` | `80` | Обрезка |

### `random-compare` — простые против случайного сита Крамера

**Нули ζ не используются.** Случайно выбираются `n` центров окон в `[x_lo, x_hi]`; для каждого считается плотность **настоящих** простых и сравнивается с **независимыми** реализациями модели Крамера: каждое целое `m ≥ 2` помечается независимо с вероятностью `1 / ln m` (псевдо-простые с похожей в среднем плотностью, без арифметики делимости). На каждую «вселенную» строится одно поле; по многим вселенным смотрят распределение средней плотности по тем же окнам. Выводится односторонняя эмпирическая **P**(средняя случайная &lt; средняя истинная).

| Опция | По умолчанию | Описание |
|--------|----------------|----------|
| `--n` | `60` | Число случайных центров окон |
| `--universes` | `100` | Независимых полей Крамера |
| остальное | как у `demo` | `x-lo`, `x-hi`, `half-width`, `seed` |

Пример:

```bash
python scripts/init_db.py
spectral-primes demo --n 30
spectral-primes demo-sparse --delta 300 --n-t 48 --n 30
spectral-primes permute --reps 200 --n 25
spectral-primes random-compare --n 80 --universes 120
spectral-primes curve --points 100 > u_curve.tsv
pytest -q
```

## Дизайн эксперимента (три группы)

1. Задаётся интервал `[x_lo, x_hi]` и **пул** случайных центров окон (целые).
2. На **референсной сетке** `x` в том же интервале считаются среднее `μ` и СКО `σ` выбранного оператора (`U` или `U_sparse`).
3. **Группа A:** центры, где оператор выше `μ + kσ` (по умолчанию `k = 1.5`).
4. **Группа B:** равномерно случайные центры из пула (в обычном `demo` независимо от `U`; в `permute` — **та же** фиксированная **B**).
5. **Группа C:** центры, где оператор ниже `μ − kσ`.
6. Для каждого центра оценивается **плотность простых** (простых на `10^5` целых) на `[центр − half_width, центр + half_width)`.

По умолчанию проверка простоты — **SymPy** (детерминированно при `n < 2^64`). При концах интервала ≥ 10⁹ `prime_count_interval` выдаёт **предупреждение**; можно поставить **`[fast-primes]`** (`gmpy2`) для более быстрых поштучных проверок или использовать внешнее решето при очень больших `x`. Это иллюстративный стек для умеренных масштабов, не замена промышленному решету на масштабах препринта.

## Случайное сито (модель Крамера)

Чтобы отделить «**разреженность как у простых**» от «**реальных простых**», реализован классический **стохастический нуль**: каждое `n ≥ 2` независимо помечается с вероятностью `1 / ln n` (для `n < 2` — никогда). Это согласуется с главным членом плотности `~ 1 / ln x`, но **игнорирует делимость** (модель сознательно грубая).

Команда `random-compare` использует **одни и те же** случайные окна для истинных простых и для многих независимых реализаций Крамера. Высокая эмпирическая P-value означает, что средняя плотность простых в этих окнах **не выбивается** из нуля; низкая — что простые **систематически плотнее** (или реже — в зависимости от хвоста), чем случайное сито, то есть «отпечаток» сильнее, чем у независимого прореживания. Для тонкой статистики модель ненадёжна; трактуйте вывод как **разведочный**.

## Разрежённый оператор, Varₙ и перестановочные тесты

**Локальная энергия.** На `[x − δ, x + δ]` приближают

$$
E_n(x) \approx \text{mean}_t\,\bigl(r_n\,\omega(\gamma_n,\Lambda)\,\cos(\gamma_n \ln t)/\gamma_n\bigr)^2
$$

на равномерной сетке по `t`. **Относительные веса** `Var_n = E_n / \sum_k E_k`. **Подмножество** `S`: индексы с `Var_n > 1/N` (с запасным правилом, если ни один не подходит). **Разрежённое значение в точке `x`:**

$$
U_{\text{sparse}}(x) = \sum_{n \in S} r_n\,\omega(\gamma_n,\Lambda)\,\frac{\cos(\gamma_n \ln x)}{\gamma_n}.
$$

**Ранговый множитель.** `r_n = exp(rank_anneal · (slot_n+1)/N))` зависит от **позиции** нуля во входном списке (`slot_n`), а не только от `γ_n`. При `rank_anneal = 0` выполняется `r_n ≡ 1` (ближе всего к правилу «только γ» из препринта).

**Зачем `rank_anneal` в `permute`.** Если в `E_n` участвует только `γ_n` и амплитуда одинакова для всех слотов, то **перестановка γ по слотам не меняет** `U_sparse(x)` (мультимножество дающих вклад частот то же). Перестановочный нуль был бы **вырожденным**. Небольшой `rank_anneal > 0` ломает эту симметрию, чтобы перестановки меняли оператор; это **технический приём для симуляций**, а не утверждение, что в `.docx` есть такой множитель. **Полная** сумма `U(x,Λ)` при перестановке γ **инвариантна** (коммутативная сумма); см. `tests/test_subset.py`. Библиотека **отклоняет** вызов `permutation_test_sparse(..., rank_anneal=0)` без `allow_degenerate_null=True`; в CLI для `--rank-anneal 0` нужен флаг **`--allow-degenerate-null`**.

## Ограничения и честные замечания

- **Масштаб:** в препринте эффекты для очень больших `x` и многих нулей; при малых `x` и мало нулей — **шум** и другой масштаб плотности простых.
- **Статистика:** Z и p-value из CLI — **иллюстративны**; это не полное воспроизведение зарегистрированного в работе протокола.
- **Перестановки:** интерпретация зависит от `rank_anneal` и от схемы с фиксированным пулом и фиксированной **B**.
- **Модель Крамера:** удобный нуль для плотности, не полная модель корреляций простых.
- **Никаких крипто-обещаний:** это не инструмент факторизации и не криптоанализ.

## C++ MPFR tool (`spectral_u`)

В каталоге [`cpp/`](cpp/) собирается небольшая программа, которая считает **ту же** полную сумму `U(x,Λ)`, что и Python, с **произвольной точностью** и корректным округлением через [GNU MPFR](https://www.mpfr.org/) (на базе [GMP](https://gmplib.org/)).

```bash
cmake -S cpp -B cpp/build -DCMAKE_BUILD_TYPE=Release
cmake --build cpp/build
./cpp/build/spectral_u --prec 512 --lambda 80 --x 1e14
```

Зависимости, Windows / vcpkg / MSYS2 и ключи командной строки — в **[`cpp/README.md`](cpp/README.md)** (на английском). Поиск нулей дзета в C++ **не** реализован: по-прежнему `scripts/build_zeros_csv.py` и Python/`mpmath`.
